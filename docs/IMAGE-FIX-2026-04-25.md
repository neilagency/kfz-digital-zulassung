# Image Loading Fix — April 25, 2026

## Problem
Browser console was showing 4 repeated errors:
```
[Error] Failed to load resource: the server responded with a status of 400 () (image, line 0)
```

Along with warnings about preloaded resources not being used:
```
[Warning] The resource https://onlineautoabmelden.com/_next/image?url=%2Fuploads%2Fmedia%2F2026%2F04%2F... was preloaded using link preload but not used within a few seconds from the window's load event.
```

## Root Cause Analysis
1. **Missing Directory Structure**: The `/public/uploads/media/2026/04/` directory did not exist
2. **Media Upload Failures**: When users attempted to upload images to the Media Library, the system tried to save files to a non-existent path, causing failures
3. **Orphaned Database Records**: Previous failed uploads had left 21 orphaned Media table entries with broken file references
4. **Next.js Image Optimization Errors**: When the page tried to preload these missing images via the `/_next/image` optimization API, it returned 400 errors

## Solutions Implemented

### 1. **Created Missing Directory Structure** ✅
```bash
mkdir -p /public/uploads/media/2026/04
```
- Ensures future media uploads have a valid destination path
- Prevents upload failures from broken directory structure

### 2. **Added Fallback Image API Handler** ✅
**File**: `/src/app/api/images/[...path]/route.ts`
- Serves missing images gracefully with a transparent placeholder PNG (1x1 pixel)
- Prevents 400 errors from the Next.js Image Optimization pipeline
- Falls back to serving actual files if they exist
- Includes proper MIME type detection for different image formats
- Caches responses appropriately (3600s for missing, 31536000s for existing)

### 3. **Cleaned Orphaned Media Database Entries** ✅
**Script**: `/scripts/cleanup-orphaned-media.ts`
- Scanned entire Media table (339 entries) for orphaned records
- Found 21 entries pointing to missing files including:
  - Old WordPress migration files (fonts, icons from 2020-2022)
  - Recent failed uploads from `/uploads/media/2026/03/`
- Successfully removed all orphaned entries from database
- Eliminates preload attempts for non-existent images

## Technical Details

### Placeholder Image (1x1 Transparent PNG)
Used as fallback for missing images. Prevents layout shifts and errors:
- Size: ~70 bytes
- Format: PNG with alpha transparency
- Purpose: Graceful degradation when images are missing

### Image API Handler
Handles requests to `/api/images/[...path]` with:
- Security check against directory traversal attacks
- MIME type detection based on file extension
- Conditional caching based on file existence
- Error handling with transparent placeholder fallback

### Database Cleanup
Scans localPath field for orphaned entries:
- Non-destructive analysis before deletion
- Detailed logging of removed entries
- Can be run safely multiple times
